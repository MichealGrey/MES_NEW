# MES系统边界判定规则

## 核心原则

**MES的核心职责** = 生产执行(WIP) + 批次控制 + 质量采集 + 追溯 + 设备状态监控 + 物料/载具跟踪

**MES不做的** = 订单创建、主数据维护、配方设计、设备底层通讯、专业良率分析、设备维护计划

---

## 判定流程

当遇到一个新功能或菜单项时，按以下顺序判断：

### Step 1: 这是生产执行相关的吗？
- ✅ 是 → 可能是MES
  - WIP跟踪、批次操作、进站/出站、拆批/合批 → **MES**
  - 工单创建、订单管理 → **ERP**
  - 客户信息、交付计划 → **ERP/CRM**

### Step 2: 这是工艺/配方设计吗？
- ✅ 是 → 看是设计还是执行
  - 配方设计、参数定义、审批流程 → **PLM**
  - 配方下发到设备、配方执行控制 → **MES**
  - 配方与设备绑定 → **MES**

### Step 3: 这是设备通讯/控制吗？
- ✅ 是 → 看是通讯还是业务
  - SECS/GEM通讯、设备指令下发 → **EAP**
  - 设备状态展示、设备告警展示 → **MES(展示层)**
  - 设备底层报警采集 → **EAP(采集层)**

### Step 4: 这是质量相关吗？
- ✅ 是 → 看是监控还是分析
  - SPC控制图、OOC事件、FDC实时监控 → **MES**
  - 专业良率分析、Yield Learning → **YMS**
  - Wafer Map展示(基础) → **MES**
  - Wafer Map深度分析 → **YMS**

### Step 5: 这是设备维护相关吗？
- ✅ 是 → 看是计划还是执行
  - 预防性维护计划(PM Schedule) → **EAM/CMMS**
  - PM执行协调、设备状态联动 → **MES**
  - 维修工单创建 → **EAM**
  - 设备状态对生产的影响 → **MES**

## 功能归属矩阵

### 来自 ERP 系统的功能
| 功能 | 归属 | 说明 |
|---|---|---|
| 工单管理 | ERP→MES | ERP创建/下发，MES执行跟踪 |
| 主数据管理 | ERP | 物料、BOM、客户信息 |
| 工艺路线 | PLM/ERP | 工艺设计在PLM，MES执行 |
| 客户交付跟踪 | ERP/CRM | 客户订单来自ERP，MES反馈进度 |
| 生产报表(订单维度) | ERP | 基于订单的统计归ERP |
| 生产报表(MES维度) | MES | WIP、效率等归MES |

### 来自 PLM 系统的功能
| 功能 | 归属 | 说明 |
|---|---|---|
| 配方设计 | PLM | 配方创建、版本管理 |
| 配方审批 | PLM | 审批流程在PLM |
| 配方参数定义 | PLM | 工艺参数由PLM维护 |
| 配方下发 | PLM→MES | 审批后下发到MES |
| 配方执行 | MES | MES控制配方在设备上执行 |
| 设备绑定 | MES | MES将配方绑定到具体设备 |

### 来自 EAP 系统的功能
| 功能 | 归属 | 说明 |
|---|---|---|
| SECS/GEM通讯 | EAP | 设备协议通讯 |
| 设备指令下发 | EAP | Start/Stop/Abort等 |
| 设备报警采集 | EAP | 底层报警采集和标准化 |
| 设备状态监控 | MES | EAP提供数据，MES展示和业务联动 |
| 设备告警展示 | MES | 基于EAP数据做业务展示 |
| EAP控制界面 | MES | MES提供操作入口，EAP执行 |

### 来自 YMS 系统的功能
| 功能 | 归属 | 说明 |
|---|---|---|
| 良率看板 | MES | 基础良率统计 |
| 良率趋势 | MES | 趋势图展示 |
| Wafer Map(基础) | MES | 原始Map展示 |
| 良率深度分析 | YMS | 根因分析、Yield Learning |
| Bin Map深度分析 | YMS | 复杂模式识别 |
| 缺陷分类 | YMS | Defect Pareto等 |

### 来自 EAM/CMMS 系统的功能
| 功能 | 归属 | 说明 |
|---|---|---|
| PM计划 | EAM | 维护计划制定 |
| PM工单 | EAM | 维修工单管理 |
| 备件管理 | EAM | 备品备件库存 |
| 设备保养协调 | MES | 与生产计划协调 |
| 设备状态影响 | MES | PM对WIP的影响 |

## 使用示例

### 场景1: 用户问"工单管理应该放在哪？"
→ 判定: 工单创建在ERP，工单执行跟踪在MES
→ 结论: 工单管理的数据来源是ERP，MES负责执行状态反馈

### 场景2: 用户问"要不要在MES里做配方设计？"
→ 判定: 配方设计是工艺设计范畴
→ 结论: 配方设计应在PLM，MES只做配方下发和执行控制

### 场景3: 用户问"设备报警谁来管？"
→ 判定: 报警采集是EAP，报警业务展示是MES
→ 结论: EAP负责采集和标准化，MES负责业务展示和联动

### 场景4: 用户问"良率分析做不做？"
→ 判定: 基础良率统计是MES，深度分析是YMS
→ 结论: MES做基础看板和趋势，复杂分析交给YMS

## 快速判定口诀

| 特征 | 归属系统 |
|---|---|
| "创建"、"维护"、"设计" | 通常是ERP/PLM |
| "执行"、"跟踪"、"控制" | 通常是MES |
| "采集"、"通讯"、"协议" | 通常是EAP |
| "分析"、"学习"、"优化" | 通常是YMS/专业系统 |
| "计划"、"维护"、"保养" | 通常是EAM |
| "展示"、"看板"、"报表" | MES(基于其他系统数据) |
