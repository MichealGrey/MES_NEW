## 一、文档说明

### 1.1 文档目的

本文档以OSAT半导体封测厂实际业务场景为背景，从不同角色用户的真实需求出发，系统梳理从客户订单到产品出货的完整生产流程中的MES系统功能需求。

### 1.2 需求来源角色

| 角色 | 所属部门 | 文档中体现为 |
|------|---------|-------------|
| 销售/客户经理 | 商务/销售部 | REQ需求来源标注「销售」 |
| 生产计划员 | 生产计划部 | REQ需求来源标注「计划」 |
| 车间操作员/技术员 | 生产部 | REQ需求来源标注「操作」 |
| 班组长/车间主管 | 生产部 | REQ需求来源标注「管理」 |
| 工艺工程师/产品工程师 | 工程/工艺部 | REQ需求来源标注「工程」 |
| 质量工程师/IPQC/CQE | 质量部 | REQ需求来源标注「质量」 |
| 设备工程师 | 设备部 | REQ需求来源标注「设备」 |
| 仓库/物料管理员 | 供应链/仓储部 | REQ需求来源标注「物料」 |
| 管理层/总监 | 总经办/各VP | REQ需求来源标注「决策」 |

### 1.3 需求格式说明

本文档采用多种格式呈现需求：

- **用户故事 (US)：** As a ... I want ... So that ...
- **问题驱动 (PD)：** 问题描述 + 期望方案
- **痛点分析 (PA)：** 痛点分析 + 需求定义
- **场景模拟 (SS)：** 场景描述 + 功能诉求
- **合规驱动 (CD)：** 合规/标准要求 + 需求定义
- **客户定制 (CC)：** 客户特殊要求 + 需求定义
- **紧急修复 (EF)：** 紧急场景 + 快速解决方案需求
- **效率优化 (EO)：** 当前瓶颈 + 优化目标与功能需求

### 1.4 需求编号规则

```
REQ-{阶段编号}-{类型}-{流水号}
例如：REQ-03-US-001 → 减薄切割阶段 用户故事类型 第001号
```

阶段编号：01=订单, 02=来料, 03=减薄切割, 04=贴片, 05=键合, 06=塑封, 07=电镀, 08=打标切筋, 09=测试, 10=终包, 11=出货, 99=跨阶段

类型：US=用户故事, PD=问题驱动, PA=痛点分析, SS=场景模拟, CD=合规驱动, CC=客户定制, EF=紧急修复, EO=效率优化

### 1.5 优先级定义

| 优先级 | 级别 | 说明 |
|--------|------|------|
| P0 | 紧急 | 不实现则生产无法进行或严重合规风险 |
| P1 | 高 | 核心业务需求，直接影响效率或质量 |
| P2 | 中 | 改善型需求，能显著提升效率或降低风险 |
| P3 | 低 | 锦上添花型需求，提升用户体验 |

## 三、阶段1 客户订单接收

> **流程节点：** 客户下单 → 订单评审 → 工艺可行性评估 → 产能确认 → 订单接收 → 工单创建

---

### REQ-01-SS-001 【场景模拟】战略客户紧急插单

| 项目 | 内容 |
|------|------|
| **需求来源** | 销售/大客户经理 |
| **优先级** | P1 |
| **需求描述** | 作为大客户经理，我需要在系统中快速识别战略客户（Tier 1）的紧急插单请求，自动检查该客户的专属产能预留情况，并一键生成插单评估报告（含影响分析、交付延迟预估），以便在30分钟内回复客户是否接受及承诺交期。 |
| **业务背景** | 某Tier 1客户临时增加200万颗QFN封装急单，当前产线已满负荷。销售需要在极短时间内评估能否接单、何时交付，同时需要知道插单会影响哪些其他客户的工单。 |
| **关联生产阶段** | 订单接收 |
| **验收标准** | 1. 支持按客户等级自动标记订单优先级<br>2. 插单操作后自动输出受影响的工单清单及影响程度（高/中/低）<br>3. 自动生成插单评估报告（PDF格式），包含产能分析、物料齐套率、交期预估<br>4. 支持客户SLA标准展示（Tier 1 = 24h响应）<br>5. 全流程操作留痕，记录插单审批链条 |

---

### REQ-01-US-001 【用户故事】多客户工艺能力匹配

| 项目 | 内容 |
|------|------|
| **需求来源** | 销售/产品工程师 |
| **优先级** | P1 |
| **需求描述** | As a 销售人员，I want 在录入订单时系统自动校验我厂是否具备该产品的封装工艺能力，So that 我不会接下单后无法生产的产品。 |
| **业务背景** | 工厂目前支持QFN、BGA、SOP、TO、DFN等封装类型。不同客户的同一封装类型可能有特殊的工艺要求（如特殊胶水、铜线替代金线、特定塑封料）。 |
| **关联生产阶段** | 订单接收 |
| **验收标准** | 1. 录入订单时自动匹配封装类型库<br>2. 对超出当前工艺能力的产品自动弹出警告<br>3. 显示当前支持的封装类型及对应的工艺路线模板<br>4. 对特殊工艺要求标记为「需工程评估」 |

---

### REQ-01-CC-001 【客户定制】客户指定工艺参数

| 项目 | 内容 |
|------|------|
| **需求来源** | 质量/CQE |
| **优先级** | P2 |
| **需求描述** | 部分客户在订单中指定了特定的工艺参数（如焊线拉力要求>8g而非标准>6g，或塑封空洞率要求<1%而非标准<3%）。系统需在订单接收时自动提取这些客户定制参数，关联到该工单的工艺路线中，并在对应工序强制执行客户标准。 |
| **业务背景** | 某汽车电子客户要求所有产品焊线拉力从标准6g提升至8g，该要求必须贯穿所有相关工单。人工传递容易遗漏，导致批量质量事故。 |
| **关联生产阶段** | 订单接收 |
| **验收标准** | 1. 支持在订单中录入客户特殊工艺要求<br>2. 客户参数自动关联到工艺路线参数库<br>3. 客户参数与工厂标准参数并存且冲突时以客户参数为准<br>4. 工序执行时自动加载客户参数并强制校验<br>5. 客户参数变更需走ECO流程并通知相关方 |

---

### REQ-01-PD-001 【问题驱动】订单评审流程缺失线上化

| 项目 | 内容 |
|------|------|
| **需求来源** | 生产计划/工程 |
| **优先级** | P0 |
| **问题描述** | 当前订单评审完全依赖线下会议和邮件，跨部门（销售/工程/质量/生产/计划）评审周期长（平均3-5天），评审记录无法追溯，经常出现评审结论未被执行的情况。 |
| **期望方案** | 建立线上订单评审工作流，支持：<br>1. 订单自动触发评审任务分配<br>2. 各角色在线评审并填写意见<br>3. 评审意见自动汇总，支持通过/驳回/有条件通过<br>4. 评审通过后自动生成工艺路线和工单<br>5. 超时未评审自动升级提醒 |
| **关联生产阶段** | 订单接收 |
| **验收标准** | 1. 评审流程可配置（不同客户/产品/封装类型的评审路径不同）<br>2. 支持会签和串行审批两种模式<br>3. 评审超时（24h）自动提醒，48h未处理自动升级至主管<br>4. 所有评审意见、签名、时间戳不可篡改<br>5. 评审通过后自动流转到工单创建 |

---

### REQ-01-PA-001 【痛点分析】订单版本变更混乱

| 项目 | 内容 |
|------|------|
| **需求来源** | 生产计划 |
| **优先级** | P2 |
| **痛点分析** | 客户经常变更订单数量、交期或封装规格。当前没有版本管理机制，导致生产按旧版本执行、仓库按新版本备料、财务按中间版本结算，三方数据不一致。 |
| **需求定义** | 订单变更必须走版本管理流程，每次变更生成新版本号，明确标注变更内容，通知所有相关方。 |
| **关联生产阶段** | 订单接收 |
| **验收标准** | 1. 订单每次变更自动生成新版本（V1.0 → V1.1 → V1.2）<br>2. 变更内容以Diff方式展示（新增/修改/删除）<br>3. 变更自动触发通知（计划/仓库/生产/质量）<br>4. 已开工工单关联旧版本，新工单关联新版本<br>5. 保留完整变更历史，可回溯任意版本 |

---

### REQ-01-EO-001 【效率优化】批量订单快速录入

| 项目 | 内容 |
|------|------|
| **需求来源** | 销售/计划 |
| **优先级** | P3 |
| **当前瓶颈** | 客户通常批量下单（单次20-50个工单），当前需逐个录入，单次操作耗时30-60分钟。 |
| **优化目标** | 支持Excel/CSV模板批量导入订单，系统自动校验格式、工艺匹配性，校验通过后批量生成订单。 |
| **关联生产阶段** | 订单接收 |
| **验收标准** | 1. 提供标准订单导入模板下载<br>2. 导入前自动校验数据格式和必填项<br>3. 批量校验工艺匹配性，不匹配项高亮标注<br>4. 校验结果汇总报告（通过数/失败数/失败原因）<br>5. 支持部分导入（跳过失败项，导入通过项） |

## 四、阶段2 晶圆来料接收

> **流程节点：** 来料预约 → 卸货登记 → IQC检验 → 检验判定 → 入库上架 → 批次追溯建档

---

### REQ-02-US-001 【用户故事】来料批次全程追溯

| 项目 | 内容 |
|------|------|
| **需求来源** | 质量/IQC |
| **优先级** | P1 |
| **需求描述** | As a IQC检验员，I want 在来料接收时自动生成包含供应商信息、来料批次号、检验结果的追溯档案，So that 后续任何质量问题都可以追溯到具体的来料批次。 |
| **业务背景** | 晶圆来料是生产的最核心物料，每批晶圆包含多个Lot，每个Lot对应不同客户。如果某批晶圆存在缺陷，必须能快速定位使用了该批晶圆的所有在制品和成品。 |
| **关联生产阶段** | 晶圆来料接收 |
| **验收标准** | 1. 扫描供应商来料条码自动建档<br>2. 档案包含：供应商、物料编码、批次号、数量、晶圆数量、来料日期<br>3. IQC检验结果自动关联至批次档案<br>4. 支持从批次正向查询所有使用该批料的工单<br>5. 支持从来料批次反向查询至供应商出货批次 |

---

### REQ-02-CD-001 【合规驱动】晶圆MSL等级管理

| 项目 | 内容 |
|------|------|
| **需求来源** | 质量/体系管理 |
| **优先级** | P0 |
| **合规要求** | IATF 16949要求对所有湿度敏感器件进行MSL等级管理。晶圆来料必须登记MSL等级、开封日期、车间寿命( floor life )，超期物料不得使用。 |
| **需求定义** | 在来料接收环节必须录入或自动获取MSL等级，系统自动计算车间寿命倒计时，超期时强制锁定该批次。 |
| **关联生产阶段** | 晶圆来料接收 |
| **验收标准** | 1. 来料登记MSL等级（1-6级）<br>2. MSL 2级以上物料开封后自动启动车间寿命倒计时<br>3. 剩余寿命<12h时黄色预警，<4h时红色预警<br>4. 超期物料系统强制锁定，任何工序无法领用<br>5. 超期物料如需使用，需走特采审批流程 |

---

### REQ-02-PD-001 【问题驱动】供应商来料信息不完整

| 项目 | 内容 |
|------|------|
| **需求来源** | 仓库/物料管理员 |
| **优先级** | P2 |
| **问题描述** | 不同供应商的来料标签格式不统一，部分供应商缺少关键信息（如晶圆Map文件、良率报告、CP测试数据），导致IQC检验时需要人工补充信息，仓库无法准确上架。 |
| **期望方案** | 建立供应商来料信息标准化模板，来料接收时系统校验必填信息是否完整，不完整则自动触发供应商补充通知。 |
| **关联生产阶段** | 晶圆来料接收 |
| **验收标准** | 1. 支持不同供应商的来料标签模板配置<br>2. 来料接收时自动校验必填项完整性<br>3. 缺失项自动分类（关键缺失/非关键缺失）<br>4. 关键缺失自动触发供应商补件通知<br>5. 支持手动录入缺失信息并标记为「供应商待补」 |

---

### REQ-02-SS-001 【场景模拟】晶圆Map文件自动关联

| 项目 | 内容 |
|------|------|
| **需求来源** | 工艺/产品工程师 |
| **优先级** | P1 |
| **需求描述** | 作为产品工程师，我需要在晶圆来料接收时将供应商提供的晶圆Map文件（Good Die / Bad Die位置分布图）自动关联至批次档案，并确保该Map文件在减薄切割工序自动下发至切割设备。 |
| **业务背景** | 每批晶圆在出厂前经过CP测试，Map文件标记了每颗Die的良/不良位置。切割工序必须依据Map文件跳过不良Die，否则会造成浪费。当前Map文件通过U盘人工传递，容易出错。 |
| **关联生产阶段** | 晶圆来料接收 → 减薄切割 |
| **验收标准** | 1. 支持多种Map文件格式自动解析（SECS/GEM、CSV、PDF）<br>2. Map文件与来料批次自动绑定<br>3. 切割工序自动加载对应批次的Map文件<br>4. 设备实际切割结果与Map文件比对并记录偏差<br>5. 支持Map文件版本管理（供应商更新时保留历史版本） |

---

### REQ-02-PA-001 【痛点分析】来料检验周期过长

| 项目 | 内容 |
|------|------|
| **需求来源** | 质量/IQC |
| **优先级** | P2 |
| **痛点分析** | 当前IQC检验依赖纸质记录，检验完成后人工录入系统，平均检验周期2-3天。晶圆来料通常有急用需求，长检验周期导致生产线待料停工。 |
| **需求定义** | IQC检验全流程电子化，移动端PDA录入检验数据，检验结果自动判定，合格物料自动触发入库上架指令。 |
| **关联生产阶段** | 晶圆来料接收 |
| **验收标准** | 1. PDA端支持检验项目录入（外观、尺寸、电性等）<br>2. 检验结果自动与标准比对并判定合格/不合格<br>3. 合格物料自动生成入库任务并推荐库位<br>4. 不合格物料自动触发不合格品处理流程<br>5. 检验周期从接收至入库≤4小时 |
