## 一、销售与客户服务类需求

### REQ-SL-001 客户订单接收与确认

| 项目 | 内容 |
|------|------|
| **需求ID** | REQ-SL-001 |
| **需求名称** | 客户订单接收与确认 |
| **来源角色** | 销售部 / KAM |
| **需求描述** | 系统需支持从ERP自动接收客户订单，或由销售手动创建订单。订单需包含：客户编码、产品型号、封装类型、数量、交期、特殊工艺要求、测试方案代码。接收后自动触发订单确认流程。 |
| **业务场景** | 战略客户每周通过EDI下发批量订单，标准客户通过门户下单。销售需确认订单可行性（产能、物料、交期），并回复客户。 |
| **优先级** | P0-Critical |
| **生产阶段** | 订单接收 |
| **验收标准** | 1. 支持EDI/API自动接收订单，解析准确率>99.9%<br>2. 订单确认后30分钟内自动通知计划部<br>3. 支持订单变更/取消流程，记录变更历史<br>4. 支持客户特殊工艺要求字段及自动校验 |

### REQ-SL-002 客户SLA等级管理与交期承诺

| 项目 | 内容 |
|------|------|
| **需求ID** | REQ-SL-002 |
| **需求名称** | 客户SLA等级管理与交期承诺 |
| **来源角色** | 销售部 / 客服部 |
| **需求描述** | 系统需按客户层级（Tier1-Tier4）自动应用不同SLA规则。创建订单时，根据产品工艺路线、当前产能负荷、在制品排队情况，自动计算承诺交期（Commit Date）。Tier1客户订单自动标记为"紧急"，优先排产。 |
| **业务场景** | 战略客户要求24h响应、7天交付周期；标准客户72h响应、14天交付。系统需自动区分并提醒。 |
| **优先级** | P1-High |
| **生产阶段** | 订单接收 |
| **验收标准** | 1. 按客户层级自动匹配SLA规则<br>2. 交期计算准确率>95%<br>3. SLA即将超期前48h自动预警销售<br>4. 支持交期协商记录及客户确认 |
