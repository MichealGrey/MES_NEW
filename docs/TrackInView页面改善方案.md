# TrackInView 页面改善方案

## 🔍 当前页面问题分析

| 问题 | 影响 |
|------|------|
| 功能过于简陋 | 无法满足实际生产操作需求 |
| 无操作按钮 | 操作人员无法确认执行进站/出站 |
| 无批次详细信息 | 操作人员无法确认批次正确性 |
| 进站/出站模式混淆 | 容易造成操作失误 |
| 无操作历史 | 无法追溯操作记录 |
| 无条码扫描支持 | 影响操作效率 |

---

## 📋 改善方案

### 2. 核心功能改善点

| 功能项 | 说明 |
|--------|------|
| **进站/出站模式切换** | 使用RadioButton或TabControl明确区分两种操作模式 |
| **条码扫描支持** | 扫描枪自动填充，Enter键触发查询 |
| **批次信息卡片** | 展示产品型号、数量、工序、客户、优先级等关键信息 |
| **扩展校验项** | 增加载具校验、人员权限、环保材料等封测特有校验 |
| **校验状态颜色** | ✅绿色通过 / ⚠️黄色警告 / ❌红色失败 |
| **操作按钮区** | 确认、取消、强制进站（需权限） |
| **操作历史** | 显示当前操作人员最近10条操作记录 |
| **快捷键支持** | F5刷新、Enter确认、ESC取消 |

### 3. 半导体封测特有业务需求

| 业务场景 | 功能支持 |
|----------|----------|
| **分批操作** | 支持部分数量进站 |
| **合并批次** | 支持多个批次同时进站同一设备 |
| **强制进站** | 异常情况下需主管授权的强制操作 |
| **Recipe校验** | 设备配方与产品规格一致性校验 |
| **材料追溯** | 显示关键材料批号信息 |
| **环保标识** | RoHS/非RoHS产品醒目标识 |

### 4. 用户体验优化

1. **自动聚焦**：页面加载自动聚焦批次号输入框
2. **键盘操作**：Tab键切换输入框，Enter触发查询
3. **声音反馈**：操作成功/失败播放不同提示音
4. **大字体设计**：车间环境下确保可读性
5. **操作确认弹窗**：关键操作二次确认

## 🛠️ 实施步骤

### 第一步：重构XAML页面布局
- 添加进站/出站模式切换按钮
- 增加批次信息卡片区域
- 扩展校验结果列表列
- 添加操作按钮区
- 增加最近操作记录表格

### 第二步：扩展ViewModel业务逻辑
- 添加操作模式枚举（TrackIn/TrackOut）
- 增加批次详细信息属性
- 扩展校验结果模型（增加状态枚举、颜色）
- 添加操作命令（确认、取消、强制进站）
- 增加操作历史记录集合

### 第三步：样式和交互优化
- 校验结果状态颜色区分（绿/黄/红）
- 按钮状态启用/禁用逻辑
- 输入框键盘事件处理
- 弹窗确认对话框

### 第四步：业务功能实现
- 条码扫描自动填充
- 批次信息查询接口调用
- 校验规则引擎集成
- 操作日志记录

## ✅ 验收标准

1. 操作人员可通过扫描条码快速完成进站/出站操作
2. 所有关键校验项可直观展示通过/警告/失败状态
3. 异常情况下支持主管授权强制进站
4. 操作记录可追溯，显示最近10条操作
5. 支持键盘快捷键操作，减少鼠标依赖
6. 页面响应时间 < 1秒
